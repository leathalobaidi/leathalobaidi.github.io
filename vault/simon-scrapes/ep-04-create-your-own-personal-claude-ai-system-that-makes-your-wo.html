What this episode covers

Simon walks through constructing a complete personal operating system on top of Claude Code's desktop app. The foundation is two files: a CLAUDE.md that loads global behavioural rules into every conversation, and a memory.md that Claude is instructed to write to whenever you say "remember this." On top of that sits a brand-context layer — voice profile, ICP/positioning, and design tokens — each generated by a starter skill. Then come nested workstations (content, finance, ops) with their own local CLAUDE.md and memory, and client folders that mirror the same structure. Finally Simon shows the dispatch feature for firing tasks from a phone.

Concept · Skills

A Skill is an official Claude Code feature: a SKILL.md file with YAML frontmatter (name, description) plus optional bundled scripts or reference files. It packages a reusable multi-step workflow that Claude can invoke by name. The key scaling property is progressive disclosure: only the name and description sit in Claude's context until the skill is actually invoked — at that point the full body loads. This keeps your base context light even when you have dozens of skills registered. Skills are invocable via a slash command (/skill-name) or by describing what you want and letting Claude match the description.

Applied economist lens

Once the skill exists, any session in your consultancy-research workstation can invoke it with /ch-accounts-extractor and a company number — no re-explaining the extraction logic each time. That is the production-grade version of what Simon is demonstrating here.

Concept · Hierarchical CLAUDE.md (parent inheritance)

Glossary — terms from this episode

CLAUDE.md
An official Claude Code feature: a markdown file auto-loaded into context at the start of every session in that directory. The primary way to give Claude persistent instructions, project rules, and behavioural context. Hierarchical — parent-directory files cascade down.
memory.md
A human-curated markdown file that the CLAUDE.md instructs Claude to read and write to. Stores short-term working state: active projects, key decisions, contacts. Distinct from Claude Code's built-in Memory tool, which the model writes autonomously.
Skill
An official Claude Code feature: a SKILL.md file (with name/description frontmatter) that packages a reusable multi-step workflow. Only the name and description load into context until the skill is invoked — progressive disclosure keeps context light. Invoked via slash command or matching description.
Slash command
A /name shortcut for invoking a skill or pre-packaged prompt. Custom commands are markdown files in .claude/commands/. Autocompletes in the input box.
Workstation
Simon's term for a named subfolder with its own CLAUDE.md, memory.md, and resources — scoping context to a domain (content, finance, ops). Inherits global rules from the parent CLAUDE.md but adds domain-specific rules locally. Not an official Claude Code term — a folder-architecture pattern.
Brand context
Simon's collective name for three files — voice profile, ICP/positioning, and design tokens — that personalize Claude's outputs to a specific brand or client. Stored as markdown files in a brand-context/ folder.
ICP (Ideal Customer Profile)
A document specifying who your content or service is for, what they care about, and what they're against. Generated by the ICP-positioning skill and stored as ICP.md in brand-context/.
Design tokens
A structured file recording brand colours, fonts, logo rules, and visual style guidelines. Claude references this file when generating any visual output to maintain brand consistency.
Parent inheritance
The behaviour where Claude Code reads CLAUDE.md files from the current directory and all parent directories, cascading rules downward. Workstation CLAUDE.md files extend but do not replace the root CLAUDE.md.
Dispatch
A Claude mobile-app feature allowing you to fire tasks from your phone to your desktop Claude Code instance. Requires the Claude app on mobile and the desktop app running. Does not automatically load workstation context — you must specify folder and context in the task text.
Agentic operating system / Claude OS
A community framing (not an Anthropic product) for an opinionated assembly of CLAUDE.md + skills + memory + workstations into a structured personal system. The underlying features are all official; the assembly pattern is creator-driven.
Progressive disclosure (skills)
The property that only a skill's name and description sit in context at all times; the full skill body loads only when invoked. This allows registering many skills without polluting the context window.
