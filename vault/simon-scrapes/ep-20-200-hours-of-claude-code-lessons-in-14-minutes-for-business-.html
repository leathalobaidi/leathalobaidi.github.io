Concept · Skills

A skill is a SKILL.md file (with YAML frontmatter: name and description) plus any bundled reference files. It packages a reusable, multi-step workflow Claude invokes by name — for example, /companies-house-extract or /aqua-qa. The key design property is progressive disclosure: only the skill's name and description sit in Claude's context at all times; the full body only loads when the skill is invoked. This means you can have dozens of skills registered without burning your context budget — they cost nothing until you call them.

Concept · Subagents

Subagents (invoked via Claude Code's Task/Agent tool) are separate Claude instances with their own isolated context windows, spun off to handle a focused sub-task in parallel. Only the subagent's final summary returns to the main conversation. This is how you run a "review one company's accounts" task 10 times in parallel without polluting the main context. Simon's advice to start with skills first is sound — subagents add coordination overhead and are most useful once your individual skills are already stable.

Concept · MCP (Model Context Protocol)

MCP is an open standard that defines how a language model client (Claude Code) discovers and calls tools exposed by an MCP server. Concept · Context window

The context window is the finite token buffer that holds everything Claude can "see" in one session: your messages, Claude's responses, file contents it has read, tool outputs, and the loaded portions of CLAUDE.md and any active skills. Models have hard limits (Claude Sonnet 4.5 and later: 200K tokens). When the window fills, older content is either truncated or compacted — which is why instructions written early in a long session can "disappear." The /clear command resets the conversation history (but not the CLAUDE.md); /compact summarises the conversation to free space while retaining continuity.

In practice

/clear — wipe the conversation history, start fresh (CLAUDE.md reloads automatically).
/compact — summarise the current conversation to free tokens while keeping Claude oriented on the task.

Glossary — terms from this episode

Claude Code
Anthropic's agentic CLI that runs on your computer, reads/edits files, executes shell commands, and calls external tools — all from a plain-English conversation. The key distinction from Claude.ai chat: it takes actions, not just giving advice.
CLAUDE.md
A Markdown file automatically loaded into Claude's context at session start. Your one-time standing instructions: house rules, domain vocabulary, output formats, working conventions. Hierarchical: global (~/.claude/CLAUDE.md) and per-project. Author and version-control it yourself.
Prompt
The plain-English instruction you type to Claude Code. Specificity matters: a detailed, structured prompt (with format requirements, edge-case rules, output schema) consistently outperforms a vague one. In Claude Code you can also attach screenshots and URLs, not just text.
Skill
A SKILL.md file with YAML frontmatter (name, description) that packages a reusable expert workflow. Only the name and description sit in context at rest; the full body loads on invocation (progressive disclosure). The right primitive for any process you repeat regularly.
Plugin
A distributable bundle of skills, slash commands, subagent configurations, and MCP wiring — typically packaged by community creators. Not an official Anthropic product category. Treat as a starting point to inspect and adapt.
Subagent
A separate Claude instance with its own isolated context window, spawned by the main agent to handle a focused sub-task. Only its final summary returns to the main conversation. Useful for parallelising independent tasks (e.g. extracting 10 filings simultaneously) once your skills are stable.
MCP (Model Context Protocol)
An open standard for connecting Claude Code to external tools and data sources. An MCP server wraps an API and exposes it as callable functions. Community servers exist for Companies House, Gmail, Notion, Slack, SQLite, and many others. Connect only the MCPs you need for the current task — each one costs context tokens.
Context window
The finite token buffer holding everything Claude can "see" in one session: messages, file contents, tool outputs, loaded skills. When it fills with stale material, output quality degrades. The discipline: one task per session, then /clear or /compact.
/clear
A Claude Code slash command that wipes the current conversation history and reloads CLAUDE.md, giving a fully fresh context. Use it between unrelated tasks in the same working day.
/compact
A slash command that summarises the current conversation to free token space while keeping Claude oriented on the ongoing task. Useful mid-session when a long extraction run is approaching the context limit but you don't want to lose the accumulated state.
Slash command
A /name shortcut in Claude Code. Built-in commands include /clear, /compact, and /help. Custom slash commands are Markdown files in .claude/commands/ and package repeatable prompts.
