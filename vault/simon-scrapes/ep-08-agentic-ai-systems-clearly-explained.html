What this episode covers

Simon uses a single running example — content repurposing from a YouTube video — to show what the AI does at each of four levels of capability. Level 1 is a chatbot: you paste, it replies, nothing executes. Level 2 is an AI workflow (n8n/Zapier): the pipeline fires automatically but always follows the same fixed steps. Level 3 is an agentic workflow: the model chooses its own execution path inside a harness (Claude Code, Codex, Cursor) given only a goal. Level 4 is a full agentic AI system: multiple specialised skills, shared memory, MCP connections to external tools, and a deliberate human-in-the-loop review gate. Simon closes by demystifying the whole stack — it's files in folders, not magic code.

Concept · ReAct loop

Concept · Skills

In Claude Code, a skill is a SKILL.md file with a YAML frontmatter block (name, description) plus the actual instructions and any bundled reference files. Glossary — terms from this episode

Chatbot (level 1)
A conversational AI interface (ChatGPT, Claude web) that responds to input but does not execute actions or retain dynamic context between sessions. Passive; advice without action.
AI workflow (level 2)
A fixed pipeline built in automation tools (n8n, Zapier) where the AI fills defined steps but the execution path is always the same. Automated but not adaptive.
Agentic workflow (level 3)
A goal-directed run where the model decides its own execution path inside a harness. You give it the goal; it works out the steps. Claude Code is the main example.
Agentic AI system (level 4)
A multi-skill, memory-enabled, MCP-connected system that coordinates specialised agents with a human review at defined points. Production-grade; runs 95% without you.
Harness
The infrastructure around the model that gives it agency: file access, shell execution, tool calls, and the loop to iterate on its own output. Claude Code is a harness; a browser chat tab is not.
ReAct loop
Reason + Act: the inference cycle where the model reasons about the next step, takes an action (tool call, file edit), observes the result, and repeats until done. The engine inside every agentic run.
Skill
An official Claude Code feature: a SKILL.md file with a YAML header and instructions for a specific task. Only the name/description loads into context until the skill is invoked — progressive disclosure that keeps context lean.
MCP (Model Context Protocol)
An open standard for connecting external tools and data sources to a harness. An MCP server exposes named tools the model can call. Companies House, Gmail, and Notion MCPs all work this way.
Memory
Persistent context carried across sessions. The simplest form is a markdown file (CLAUDE.md) the model reads each session; more advanced forms use structured databases the model writes to. Prevents starting from zero every run.
Human in the loop
A deliberate design point where a person reviews output before it proceeds — at input (goal-setting), mid-process (phase gate), or output (quality review). Not optional in production systems; analogous to the AQuA independent-review step.
Harness engineering
Simon's term (community usage, not official Anthropic language) for the practice of assembling skills, memory, MCPs, and review gates into a level-4 system. The jump from level 3 to level 4.
OpenClaw / Hermes
Community-created open-source personal agent frameworks — opinionated assemblies of files and skills on top of Claude Code. Not Anthropic products; useful pointers to the level-4 pattern but their specifics change rapidly.
