Concept · File-based vs MCP integration

SkillUI is a standalone Node.js CLI tool — it is not an MCP server, and it does not run inside Claude Code's context. It runs in your terminal, scrapes a website using Playwright (a browser automation library), and writes a structured output to disk. Claude Code then reads that output as a file. Front-loading the design system is the primary technique for avoiding generic AI aesthetics.
Skill UI
A community skill (loaded inside Claude Code with /skill-ui) that scrapes a given website and extracts its design principles — palette, typefaces, spacing, tone — into a structured design spec. Skills
Reusable workflow packages for Claude Code: a SKILL.md file with YAML frontmatter (name, description) plus any supporting scripts. Only the name and description sit in context until the skill is invoked, then the full body loads — this is progressive disclosure, the property that lets you have many skills without bloating context. Claude Design does not yet support custom skills.
Reference screenshot method
Feeding 3–5 screenshots of websites you admire (or your own brand assets) directly into Claude Design as the first message, so the model derives a design system from real examples rather than its baked-in defaults. Inline comment editing
Claude Design's interaction mode where you click a rendered element and type a comment to change only that element, rather than re-prompting the whole page. Crucial for token efficiency: re-prompting resets context and burns usage limits; targeted comments do not.
Token budget / usage limits
Claude Design charges tokens for each generation. The hybrid workflow where Claude Code (with full skill support, MCP access, and filesystem tools) handles design-system extraction and any back-end logic, then the artefact is handed off to Claude Design for visual iteration.
