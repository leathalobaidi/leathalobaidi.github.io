Glossary — terms from this episode

n8n
An open-source workflow-automation tool with a drag-and-drop canvas interface. Excellent for building visual, debuggable internal automations; restricted by its sustainable-use licence from being used to build products sold to third parties. Separate from Claude Code — not a competitor, a complement.
MCP (Model Context Protocol)
Anthropic's open standard for connecting Claude Code to external tools and data sources via MCP servers. Each server exposes callable tools (e.g., Companies House API, Gmail, Notion) that Claude can invoke by natural-language instruction.
Skills
Reusable Claude Code workflows packaged as SKILL.md files with a YAML header. Only the name and description load into context by default; the full body loads on invocation. Lets Simon (and you) describe a complex multi-step procedure once and invoke it by name.
GSD ("Get Shit Done")
A community framework (not an official Anthropic feature) structuring Claude Code projects into discuss → plan → execute → verify phases, each with its own context document. Reduces context drift on long builds.
Ralph loop
A community pattern that re-feeds a fixed prompt.md to Claude Code each iteration until a cap is reached, enabling unattended batch runs. Useful for repetitive extraction or QA loops.
Sustainable use licence
n8n's licence term that restricts usage to internal business purposes and explicitly forbids building products sold to third parties where the value derives substantially from n8n functionality. Claude Code-produced code carries no such restriction — the code is yours.
Version propagation
The ability to update shared logic once and have that change apply across all workflows or instances that use it. Claude Code supports this naturally through standard code versioning (git, functions); n8n does not, requiring manual per-workflow edits.
Execution history log
n8n's visual per-run inspection interface: every workflow run is stored, every node's input and output is click-inspectable in plain language. Simon identifies this as n8n's most durable advantage over Claude Code — non-technical teams can debug without reading code or logs.
Multi-tenant
A system architecture where multiple clients each have isolated data and potentially isolated interfaces within a single hosted codebase. n8n's licence prohibits this use case; Claude Code can build it freely because you own the output code.
RAG (Retrieval-Augmented Generation)
A pattern where a language model is augmented with retrieved documents or data at query time, improving accuracy on specific knowledge domains. Mentioned as one of n8n's listed use cases; not the main focus of this episode.
