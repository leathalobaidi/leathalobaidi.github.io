Concept · /loop and cron jobs

/loop is an official Claude Code slash command that takes a natural-language prompt and an interval, then registers a cron job internally so Claude re-runs that prompt on a timer without further user input. Under the hood Claude Code exposes three cron tools — cron_create, cron_list, and cron_delete. "Cron" stands for "command run on notice"; the schedule is expressed in standard cron syntax (five fields: minute, hour, day-of-month, month, day-of-week). The loop runs inside your current terminal session — it has no persistence beyond that session.

If we go to the Claude Code docs we can see there are three tools it uses: cron create, cron list, and cron delete — exactly as they sound. A cron is a notice for some sort of command to run at a given time, and that's exactly what it stands for. And just for quick reference, it's done using cron expressions, which effectively are different symbols for different time values. That's it.

Now let's talk about the limitations. Loops expire after 3 days — that is probably the biggest limitation so far, and it's a safety thing so you don't accidentally have 20 loops running forever. They also only live in your current session. So if you close the terminal, they're completely gone, and they don't even catch up. If the session was closed when a loop was supposed to fire, it just disappears and never runs again. So think of loops as "help me right now on this project" — watch my inbox for important emails, track changes across a sprint, anything where you need Claude checking in for the next few hours or days. It's not long-term.

Worth flagging

Loops are session-scoped — the moment you close the terminal they vanish, and missed firings are not recovered. This makes /loop unsuitable for anything that needs to survive overnight or across restarts. For those use cases the answer is /schedule (Feature 2 below). Also: the 3-day expiry cap means a loop started on Monday will be dead by Thursday regardless of whether you've kept the session open.

Glossary — terms from this episode

/loop
Official Claude Code slash command that registers a cron job to re-run a natural-language prompt on a recurring interval within the current terminal session. Expires after 3 days; missed firings are not recovered when the session is closed.
/schedule
Persistent scheduled task (set via Claude Code desktop app or co-working UI) that starts a fresh agent instance on a cron schedule and survives session close. Missed runs are caught up on next app launch. At time of recording: desktop app only, not terminal CLI.
Cron job / cron expression
A standard Unix scheduling mechanism. A cron expression is five whitespace-separated fields (minute, hour, day-of-month, month, day-of-week) that define when a command fires. Claude Code uses cron_create, cron_list, and cron_delete tools internally to manage loop and schedule entries.
Google Workspace CLI
Open-source command-line tool from Google (beta at time of recording) that exposes 100+ shell commands covering Drive, Docs, Sheets, Slides, Gmail, and Calendar. Claude invokes it via the Bash tool; no REST API integration required. Creates natively formatted Google documents rather than raw markdown.
Skills 2.0
Simon's shorthand for the updated Anthropic Skill Creator skill, which now includes an evaluation harness that runs a skill N times in parallel using subagents, scores each output against user-defined criteria, and returns a structured HTML report. Not an official Anthropic product name.
Skill Creator skill
A meta-skill built and maintained by Anthropic that helps users author, structure, and (after the 2.0 update) evaluate other Claude Code skills. It loads when invoked and uses the full skill scaffolding — YAML frontmatter, reference files, step-by-step process — to guide skill construction.
Subagent (parallel)
An agent spawned with its own isolated context window and tool set. In the eval harness, the orchestrator spawns one subagent per test run; all five run simultaneously and return their outputs and scores to the orchestrator for aggregation. Each subagent is independent — they do not share state.
AB test (skills)
Running the same eval with two variants — e.g., skill vs no-skill, or full reference-file set vs trimmed set — then comparing scores, token counts, and timing side by side. Used to objectively determine whether a skill addition improves output quality enough to justify its cost.
Progressive disclosure (skills)
Only the name and description fields of a skill's YAML frontmatter sit in Claude's context by default; the full SKILL.md body loads only when the skill is explicitly invoked. This keeps the base context window lean even across large skill libraries.
Cron expression fields
Five fields: minute hour day-of-month month day-of-week. Each can be a number, * (any), */n (every n), or a range. E.g., 0 9 * * 1 = 09:00 every Monday. Claude Code accepts natural language (e.g., "every Monday at 9am") and converts it internally.
