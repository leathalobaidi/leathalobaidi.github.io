Concept · .env files and API credentials

A .env file is a plain-text file in a project root that holds environment variable definitions (e.g. OPENAI_API_KEY=sk-...). Shell processes and Claude Code read these at startup; the values are never committed to git. Because the file starts with a dot, most operating-system UIs hide it by default — which is exactly why the Claude desktop file browser can't show it yet. For now, edit it in a terminal or VS Code when you need to add or rotate credentials.

Glossary — terms from this episode

Claude desktop app
The official GUI application for Claude, distinct from the web interface and the terminal-based Claude Code CLI. As of April 2026, it runs Claude Code natively and now exposes a file browser, plan sidebar, and split view.
Agentic OS (agentic operating system)
Simon's community term for a deliberate folder structure — with CLAUDE.md, a context folder, a skills folder, and an output folder — that acts as persistent memory and instruction set for Claude Code sessions. Not an official Anthropic product; treat as a design pattern.
CLAUDE.md
A markdown file Claude Code reads automatically when it starts a session inside a directory. Used to store persistent instructions, project schemas, QA rules, and style guides that should apply throughout the session.
File sidebar
The new panel in Claude desktop that shows the folder tree of the working directory and renders selected files (especially markdown) in a preview pane, without requiring a separate editor.
Plan view / plan sidebar
A panel that renders Claude's structured markdown plan alongside the chat, so you can track what the agent committed to doing without scrolling back through the conversation.
Split view
The ability to display two, three, or four independent Claude sessions tiled on screen simultaneously. Each pane has its own context, history, and tool calls — they do not share state.
Multiple sessions
Running more than one Claude Code conversation concurrently. Anthropic's April 2026 update formalised this in the desktop UI with a holistic session list and the ability to relate sessions to projects.
.env file
A hidden plain-text file (starts with a dot, so invisible in most UIs) that holds environment variable definitions including API keys. Claude Code reads it at startup but the April 2026 desktop app cannot display or edit it yet — use a terminal or VS Code for that.
Hidden files / dotfiles
Files and folders whose names start with a dot (e.g. .claude, .env). Most file browsers hide them by default. The Claude desktop file browser currently omits them, which is why the skills folder under .claude is not yet visible or editable from the UI.
Context management
The practice of controlling what information is loaded into the model's context window for a given session — by choosing the right starting directory, writing a focused CLAUDE.md, and scoping output folders — so the model has what it needs without being overwhelmed by irrelevant content.
