What this episode covers

Simon demos a Claude Code skill system he calls the "social content creator" that turns any input — a topic, a video, a URL, an audio file, a PDF — into platform-ready LinkedIn or Instagram carousels that match your brand. The episode explains why generic AI carousels fail (no design system), then walks through onboarding (brand voice profiling, visual identity tokens, layout templates), the multi-scenario content pipeline (trending research, designer subagent, image generator agent), and a final Canva Magic Layers step to make the 95%-good output fully editable. Simon's carousel system is a worked example of the most transferable Claude Code pattern: capture domain rules once in a config/token file, then have every run read from that single source of truth rather than re-inferring from scratch. The episode also shows a clean onboarding → generate → human-review → publish workflow that maps directly onto a repeatable analytical pipeline.

Concept · Skills and skill systems

In Claude Code, a skill is a reusable workflow packaged in a SKILL.md file with YAML frontmatter naming it. Only the name and description sit in context at startup; the full body loads only when the skill is invoked — this is progressive disclosure, the key scaling property. A skill system (Simon's term) is an opinionated assembly of multiple skills, slash commands, subagents, config files, and CLAUDE.md instructions working together as a pipeline. This is community framing for a structured personal setup, not a named Anthropic product.

Applied economist lens

Concept · Onboarding skills and progressive setup

Simon's onboarding is itself a skill — it runs once, writes a persistent config file, and then subsequent invocations of the content-generation skill read from that file and skip the onboarding path. Concept · Skill chaining

Skill chaining is the pattern where one skill invokes another as part of its workflow — here, the social content creator skill invokes the "trending research" skill as a sub-step when the input is a topic rather than a ready-made source. This is a usage pattern, not a named Claude Code feature: it works because invoking a skill just loads its instructions into context and runs them. The practical value is modularity — the research skill can also be called standalone, and the content skill doesn't need to duplicate research logic.

Concept · Subagents and the Task tool

A subagent in Claude Code is a separate agent instance spawned via the built-in Task (Agent) tool. It gets its own isolated context window and its own tool set — allowing it to work on a focused sub-task (here: visual inventory and slide planning) without polluting the main agent's context. Only the subagent's final output returns to the orchestrator. Simon's "designer subagent" and "image generator agent" are two such specialised spawned agents — each receives only the context it needs, which keeps their reasoning clean and avoids context rot.

Concept · Human-in-the-loop

A human-in-the-loop step is a deliberate pause in an agentic pipeline where the agent presents its interim output to the human for approval before proceeding. In Claude Code this is implemented by the skill's instructions telling the agent to stop and ask — using the built-in "ask user questions" behaviour — rather than continuing autonomously. Simon inserts this between slide planning and image generation: you see the slide plan, judge whether it represents real value for your audience, and only then let the image generation proceed.

Skill chaining

Subagent
An agent spawned via Claude Code's Task (Agent) tool with its own isolated context window and tool set; only its summary returns to the main agent. Used here for the designer agent and the image generator agent separately.
Human-in-the-loop
A deliberate pause in an agentic pipeline where the agent presents interim output for human review before proceeding. Implemented by skill instructions telling the agent to stop and ask — not automated.
Config file / tokens
A persistent file (JSON, markdown) written during onboarding that captures domain rules — visual identity tokens, brand voice parameters, platform defaults — and read by every subsequent run to enforce consistency.
Progressive disclosure
The property of skills whereby only the name and description sit in context until a skill is invoked; the full body loads only when needed. Keeps the base context window small even across large skill libraries.
Visual identity tokens
Simon's term for the specific hex codes, typeface names, layout grid values, and template references saved in the config. Analogous to a data dictionary: a precise, machine-readable spec the agent applies deterministically.
Agentic operating system

Trending research skill

95% draft

Vibe coding
