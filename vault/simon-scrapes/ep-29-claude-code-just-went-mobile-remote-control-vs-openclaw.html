Concept · Remote control (claude remote)

Remote control is a first-party Anthropic feature that exposes your running Claude Code terminal session over a web interface. You run claude remote in your terminal and Claude Code prints a URL and an optional QR code. Anyone who scans that QR code — on any device — gets a browser window wired into the same session, with full access to the same working directory, skills, agents, and slash commands. The session lives on the host machine; the phone is just a thin window. Available on Claude Pro and Max plans (from around $20/month).

Concept · CLAUDE.md

CLAUDE.md is a markdown file, typically at the root of a project, that Claude Code reads automatically at the start of every session. Concept · Permission prompts on mobile

Claude Code's permission system requires explicit approval before any action that writes to disk, runs shell commands, or calls external APIs — by default, it pauses and asks. Remote control surfaces these prompts in the mobile browser window, so you can approve or deny them from your phone. The --dangerously-skip-permissions flag that power users sometimes pass to bypass all prompts is not available in remote control mode — a deliberate safety constraint for mobile sessions.

Glossary — terms from this episode

Remote control (claude remote)
A first-party Anthropic feature that exposes a running Claude Code terminal session in a browser via a QR code or URL. The host machine does the work; the phone is a thin control window. Available on Pro and Max plans.
OpenClaw
An open-source, community-built platform (not an Anthropic product) that wraps an AI model into a 24/7 messaging-based task agent, typically self-hosted on a Mac Mini or VPS. Model-agnostic, but carries significant security surface area from community skills.
CLAUDE.md
A markdown file Claude Code reads automatically at session start, containing project conventions, schema, constraints, and ongoing tasks. Persists context across sessions without re-prompting.
Permission prompt
Claude Code's pause-and-ask mechanism before any write, shell, or external-API action. On mobile remote control, these appear in the browser window for you to approve or deny.
--dangerously-skip-permissions
A flag that bypasses all permission prompts, allowing fully unattended runs. Deliberately unavailable in remote-control mode — you must approve permissions from the mobile window.
/init
A slash command that instructs Claude Code to inspect the current repository and write a CLAUDE.md summarising its structure and conventions. Useful at the start of a new remote session to orient Claude to the project.
VPS (Virtual Private Server)
A rented remote Linux machine running 24/7 in a data centre. Hosting Claude Code on a VPS means the session is always-on and never drops due to laptop sleep — replicating OpenClaw's always-available benefit with a first-party tool.
Session window (10 minutes)
If the host machine's Wi-Fi drops or the laptop sleeps, the remote control session stays alive for approximately 10 minutes before dropping. The fix: prevent sleep on the host, or run on a VPS.
Skill
A reusable prompt package (a SKILL.md file plus optional assets) stored in a project repository that Claude Code can load and execute. Skills are accessible in remote control sessions because they live in the same directory.
Model-agnostic
Compatible with multiple underlying AI models (Claude, GPT, Ollama local models, etc.). OpenClaw is model-agnostic; Claude Code's remote control is inherently Claude-specific.
Tailscale
A zero-config VPN tool used by OpenClaw to expose a self-hosted machine's network to your phone without port-forwarding. One of several infrastructure components required to run a full OpenClaw setup.
