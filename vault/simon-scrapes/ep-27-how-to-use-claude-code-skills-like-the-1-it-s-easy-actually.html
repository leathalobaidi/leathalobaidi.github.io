Concept · Skills

A skill is a folder that packages a reusable workflow for Claude Code. It contains a SKILL.md file (the process instructions — think of it as an SOP for an AI agent) and optionally: reference files (knowledge and examples), scripts (executable code), and asset files (sample outputs). Skills are installed into ~/.claude/skills/ (global) or .claude/skills/ (project-local). Claude selects which skill to invoke by scanning only the name and description from each skill's YAML front matter — the body loads only when the skill is actually used. This is the key scaling mechanism.

Concept · Progressive disclosure

Progressive disclosure is the mechanism that lets you install many skills without bloating Claude's context. Only the YAML front matter of each skill — the name and description fields — is loaded into the initial system prompt. The full SKILL.md body is injected into context only when Claude decides to invoke that skill. Reference files and scripts are loaded only when the SKILL.md body explicitly calls for them. This staged loading is what makes the architecture scale: the menu of available skills is a compact list of labels, not a dump of all the instructions.

Concept · "Point don't dump" — selective reference loading

The best-built skills separate process (in SKILL.md) from knowledge (in reference files). The SKILL.md body doesn't contain the knowledge itself — it contains instructions about which reference file to load, and when. An SEO skill might have two reference files: title-formulas.md loaded only during title ideation, and content-structure-templates.md loaded only during content structuring. This keeps context lean even when the underlying knowledge base is large, and also separates debugging concerns: a bad output is either a process failure (fix SKILL.md) or a knowledge failure (update the reference file).

Concept · Skill activation & the 15,000-character budget

Skill activation is the process by which Claude reads the YAML front matter of every installed skill and decides whether to invoke one. Anthropic caps the combined skills list at 15,000 characters — meaning the sum of all installed skill names and descriptions must fit within that budget. When you exceed it, skills get truncated or dropped from the selection list. A well-written description uses specific, distinct trigger phrases so Claude reliably selects the right skill; vague or overlapping descriptions cause the model to guess. The 20%–84% activation-rate range Simon cites reflects how much description quality matters — even with a small, well-built library.
