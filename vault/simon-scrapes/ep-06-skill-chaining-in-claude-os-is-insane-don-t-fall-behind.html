What this episode covers

Simon argues that most people are building their Claude OS wrong in one of two ways: keeping skills isolated (acting as the manual connector) or cramming everything into one giant "mega skill" that destroys modularity, maintainability, and progressive disclosure. He walks through real examples from his own setup (video-to-short-clips, video-to-article, social carousel, slide generation) to show how shared child skills compound across multiple systems when kept modular.

Concept · Skills

Concept · Progressive disclosure

Progressive disclosure is the key scaling property of the skills architecture. Only a skill's name and description (from YAML frontmatter) load into context at startup. The full SKILL.md body — instructions, examples, reference content — only loads when the skill is invoked. A mega skill destroys this: its entire body is effectively always "loaded" once invoked, and it typically tries to do too many things in a single context, reducing the model's ability to do any of them at the level a focused skill would.

Concept · Subagents / Task tool (the orchestrator pattern)

When Claude Code spawns a subagent using the Task tool, it creates an agent with its own isolated context window and tool set. Only the subagent's final summary returns to the main agent. This isolation is the core advantage: the main orchestrator context stays clean and focused, while each subagent does deep, focused work — scraping, extracting, verifying — without polluting the orchestrator's window with all its intermediate tool outputs.

Glossary — terms from this episode

Skill system
Simon's term (community, not official Anthropic) for an orchestrator skill wired to multiple child skills that runs an end-to-end workflow. The orchestrator holds the instruction set; child skills do focused work and pass results back.
Orchestrator skill
The "brain" skill that coordinates a skill system — dispatches to child skills, receives their outputs, and sequences the overall pipeline. Mirrors the orchestrator pattern in subagent use.
Child skill
A focused, single-responsibility skill invoked by an orchestrator. Designed to be reusable across multiple skill systems.
Skill chaining
Using one skill to invoke others in sequence to accomplish a multi-step goal. A usage pattern, not a named Claude Code feature.
Mega skill (anti-pattern)
A single large skill file that tries to execute an entire end-to-end process. Destroys modularity, maintainability, and progressive disclosure — the three core advantages of the skills architecture.
Progressive disclosure
The design property of skills where only the name and description load into context at startup; the full skill body loads only when the skill is invoked. Mega skills defeat this by forcing too much context at once.
Skills
Official Claude Code feature: a SKILL.md file with YAML frontmatter (name, description) plus optional bundled files. Packages a reusable workflow Claude can invoke by name.
Subagent
An agent spawned with its own isolated context window and tools to do focused work; only its final summary returns to the main agent. The mechanism underlying the orchestrator pattern.
Claude OS / agentic OS
Community framing (not an Anthropic product) for an opinionated assembly of CLAUDE.md, skills, subagents, and commands into a structured personal system. Simon uses this term throughout; Anthropic has no product by this name.
CLAUDE.md
The memory/instructions file auto-loaded into context. Hierarchical: a user-global ~/.claude/CLAUDE.md and a per-project ./CLAUDE.md. The shared context all skill systems in your OS inherit.
Onboarding (skill)
A configuration step built into a skill system that gathers user-specific settings (paths, preferences, thresholds) once at setup, writing them into shared config that all child skills then read.
